# ТЗ #3: Аутентификация HuggingFace в расширенных настройках

## Контекст

При первом запуске приложение скачивает модели (sentence-transformers, sentiment-модель) с
HuggingFace Hub. Без авторизации HuggingFace в последнее время жёстко троттлит анонимные
загрузки (скорость падает до ~1 КБ/с, что превращает скачивание модели в ~1 ГБ в загрузку на
несколько дней). Авторизованные запросы (с personal access token) скачивают на нормальной
скорости (мегабайты/сек).

Нужно дать пользователю возможность ввести свой HuggingFace токен прямо в веб-интерфейсе
приложения — без необходимости лезть в терминал и руками выставлять переменные окружения или
запускать `huggingface-cli login`.

Это итерация поверх существующего кода — правь точечно, не переписывай файлы целиком.

---

## Требования к реализации

### 1. Backend: эндпоинты для управления токеном

Добавь в `main.py` (или новый модуль `auth_hf.py`) три эндпоинта:

- **`POST /api/hf-token`** — принимает `{"token": "hf_..."}`, сохраняет токен **локально на
  диске сервера** (не в коде, не в репозитории) в файл, например
  `backend/.hf_token` (добавь его в `.gitignore`, если ещё не там), и применяет его в текущей
  сессии через `os.environ["HF_TOKEN"] = token` и/или
  `huggingface_hub.login(token=token, add_to_git_credential=False)`
- **`GET /api/hf-token/status`** — возвращает, установлен ли токен в данный момент
  (`{"is_set": true/false, "masked_token": "hf_***...***abcd"}` — показывать только первые 3 и
  последние 4 символа токена, остальное замаскировать звёздочками, **никогда не возвращать
  токен в открытом виде через API после сохранения**)
- **`DELETE /api/hf-token`** — удаляет сохранённый токен (файл и переменную окружения),
  откат на анонимный режим
- **`POST /api/hf-token/validate`** — делает тестовый запрос к HuggingFace API
  (`GET https://huggingface.co/api/whoami-v2` с заголовком `Authorization: Bearer {token}`)
  **до** сохранения токена, чтобы сразу сказать пользователю "токен валиден, привет {username}"
  или "токен невалиден" — не дожидаясь начала реальной загрузки модели, чтобы ошибка не
  всплыла спустя 10 минут закачки

При старте приложения (`main.py`, в lifespan/startup event) — проверяй, есть ли сохранённый
файл токена, и если есть — подгружай его в окружение автоматически, чтобы пользователь не
вводил токен при каждом перезапуске сервера.

### 2. Применение токена к загрузке моделей

В модуле, где грузятся модели (`models.py` или аналогичный), убедись, что:
- `SentenceTransformer(...)`, `AutoModel.from_pretrained(...)` и любые другие вызовы,
  скачивающие веса с HuggingFace, **автоматически подхватывают токен из переменной окружения
  `HF_TOKEN`** (это поведение по умолчанию в свежих версиях `huggingface_hub`/`transformers` —
  убедись, что версии в `requirements.txt` это поддерживают, актуальные библиотеки делают это
  без доп. кода)
- Если токен не установлен — всё продолжает работать как раньше (анонимный режим, просто
  медленнее), никакого падения или обязательного требования токена

### 3. Frontend: блок в расширенных настройках

В существующий раздел "Расширенные настройки" (где уже есть toggle для Ollama/LLM-названий
тем и слайдер количества тем из прошлой итерации) добавь новый блок **"Ускорение загрузки
моделей (HuggingFace)"**:

- Короткое объяснение текстом (2-3 строки, дружелюбным языком, не сухой техникой):
  > "HuggingFace ограничивает скорость скачивания моделей без авторизации — загрузка может
  > занять часы вместо минут. Введите бесплатный токен, чтобы ускорить процесс."
  > Ссылка-текст "Получить токен →" ведущая на `https://huggingface.co/settings/tokens`
  > (открывать в новой вкладке, `target="_blank"`)
- Поле ввода токена (`type="password"`, с кнопкой "глазик" показать/скрыть текст, как в обычных
  полях пароля)
- Кнопка **"Проверить и сохранить"** — вызывает `/api/hf-token/validate`, если ок — затем
  `/api/hf-token`, показывает зелёную галочку + "Привет, {username}!" при успехе, красный крест
  + понятную причину при ошибке (невалидный токен / нет соединения с HF)
- Если токен уже сохранён (проверяется через `/api/hf-token/status` при загрузке страницы) —
  показывать замаскированный токен (`hf_***...abcd`) и кнопку **"Удалить токен"** вместо формы
  ввода
- Статус-индикатор скорости: после сохранения токена, перед стартом анализа, можно (опционально,
  не блокирующее требование) сделать мини-тест скорости — скачать маленький тестовый файл с HF
  и показать "Скорость соединения с HuggingFace: ~X МБ/с" — чтобы пользователь сразу видел эффект

### 4. Безопасность токена

- Токен **никогда** не должен попадать в логи (ни backend console output, ни browser console)
- Токен **никогда** не должен возвращаться полностью через любой API-ответ после первого
  сохранения — только маскированная версия
- Файл с токеном на диске — права доступа `chmod 600` (только владелец может читать/писать),
  выставляй это программно при сохранении файла (`os.chmod(path, 0o600)`)
- В README явно укажи: токен хранится локально в `backend/.hf_token`, нигде не передаётся
  на сторонние серверы кроме самого HuggingFace API при скачивании моделей

### 5. UX-деталь: что показывать во время реальной загрузки модели

Если в процессе скачивания модели (на экране прогресс-бара, который уже есть в приложении)
скорость загрузки **аномально низкая** (< 50 КБ/с в течение, скажем, 10+ секунд) — выводить
прямо на экране прогресса мягкую подсказку:
> "Скачивание идёт медленно — это может быть из-за отсутствия HuggingFace-токена. Можно
> прервать и настроить токен в Расширенных настройках для ускорения."
с кнопкой-ссылкой, открывающей блок настроек токена (если технически просто — открыть в новой
вкладке/модалке, не обязательно прерывать текущий процесс закачки).

Если такая детекция скорости сейчас сложно вписывается в существующий пайплайн прогресса —
это **необязательное улучшение**, не блокирующее основную задачу (эндпоинты + UI настройки
токена), сделай его при наличии времени, иначе пропусти и просто оставь статичную подсказку в
блоке настроек токена.

## Проверка перед сдачей

1. Запусти приложение без токена — убедись, что всё работает как раньше (просто медленнее
   при первой загрузке модели)
2. Введи валидный токен через UI, убедись, что:
   - Появляется подтверждение с именем пользователя
   - При перезапуске сервера (`Ctrl+C` → заново `run.sh`) токен не нужно вводить повторно
   - Скорость скачивания модели реально возрастает (если модель уже скачана — удали кэш
     `~/.cache/huggingface/hub/` для конкретной модели, чтобы протестировать с нуля, либо
     просто проверь через `/api/hf-token/validate`, что токен принимается HF API)
3. Введи невалидный токен (случайную строку) — убедись, что получаешь понятную ошибку, а не
   падение сервера
4. Удали токен через UI — убедись, что приложение откатывается в анонимный режим без ошибок

## Файлы, которые скорее всего потребуют изменений

- `backend/main.py` — новые эндпоинты, startup-хук подгрузки токена
- `backend/auth_hf.py` — новый модуль (или логику можно встроить в `main.py`, если агенту
  удобнее, но чище отдельным файлом)
- `backend/models.py` — убедиться, что токен подхватывается при загрузке моделей
- `frontend/index.html` — новый блок UI в разделе расширенных настроек
- `requirements.txt` — убедиться, что `huggingface_hub` версия актуальная (>=0.24, там точно
  стабильная поддержка `HF_TOKEN` env var и `login()`)
- `.gitignore` — добавить `backend/.hf_token`, если файл .gitignore существует в проекте
- `README.md` — короткий раздел "Как ускорить первую загрузку моделей (опционально)"
