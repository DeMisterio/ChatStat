# Telegram Chat Analyzer

Полнофункциональное веб-приложение для анализа экспортированных чатов Telegram. 
Позволяет загрузить файл `result.json` и получить детальную структурную и семантическую аналитику с помощью моделей NLP.

## Функционал
- **Структурная аналитика:** активность по времени, участники, длина сообщений, доля контента и др.
- **Семантическая аналитика:** тематическое моделирование (KMeans), анализ тональности (sentiment analysis), эмоции, словари участников и др.
- **Абсолютная приватность:** все данные обрабатываются локально. Ни один байт из переписки не уходит в сеть. Промежуточные данные и кэш анализа сохраняются локально в директории `backend/.checkpoints` (по умолчанию удаляются через 30 дней) для мгновенной повторной загрузки. Вы в любой момент можете очистить эти данные вручную прямо в интерфейсе.

## Требования
- Python 3.11+
- Около 1 ГБ ОЗУ для загрузки локальных моделей машинного обучения (`sentence-transformers`, `rubert-tiny-sentiment-balanced`). Модели скачаются автоматически при первом запуске.

### Опционально: Умные названия тем
Для генерации человекочитаемых названий тем вы можете использовать локальную LLM:
1. Установите [Ollama](https://ollama.com/) (например, на Mac: `brew install ollama`).
2. Запустите её в фоне.
3. Скачайте модель по умолчанию: `ollama pull qwen2.5:1.5b`.

## Запуск

### Быстрый запуск (Mac/Linux)
```bash
./run.sh
```

### Запуск вручную
1. Создайте и активируйте виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
2. Установите зависимости:
   ```bash
   pip install -r backend/requirements.txt
   ```
3. Запустите сервер:
   ```bash
   uvicorn backend.main:app --reload
   ```
4. Откройте в браузере: `http://localhost:8000`

## Как экспортировать чат из Telegram?
1. Откройте Telegram Desktop.
2. Откройте нужный чат (личный или групповой).
3. Нажмите на три точки в правом верхнем углу.
4. Выберите `Export chat history` (Экспорт истории чата).
5. Снимите галочки с медиафайлов для ускорения экспорта (если они не нужны).
6. **ОБЯЗАТЕЛЬНО** выберите формат **Machine-readable JSON**.
7. Нажмите Export и загрузите полученный `result.json` в анализатор.

> **Примечание об удалённых сообщениях:**
> Метрика "Удалённых сообщений (оценка по ID)" использует эвристику подсчета разрывов в последовательности `message_id`. В Telegram-экспортах разрывы ID могут возникать не только из-за удалений, но и из-за сервисных сообщений, медиа-альбомов и реакций, поэтому это число может быть завышено и является лишь грубой прикидкой.

## Ускорение загрузки моделей (Опционально)
При первом запуске приложение скачивает AI модели (sentence-transformers и sentiment) с серверов HuggingFace. Чтобы избежать ограничений скорости загрузки, вы можете использовать свой бесплатный токен `HuggingFace`.
1. Перейдите на [HuggingFace Tokens](https://huggingface.co/settings/tokens) и создайте токен.
2. В веб-интерфейсе Мега-анализатора (до загрузки файла) нажмите **⚙️ Расширенные настройки**.
3. Введите токен и нажмите **Проверить и сохранить**.
> Токен сохраняется исключительно локально в файле `backend/.hf_token` (используются строгие права доступа). Он никуда не передаётся, кроме официального API HuggingFace при скачивании моделей.

## Безопасная работа с удалённым GPU (Vast.ai)
Если вы хотите ускорить "Advanced Analysis" (ТЗ #10) с помощью мощных LLM (например, gemma4:31b) на арендованных облачных GPU, используйте систему сквозного локального шифрования файлов.

### Инструкция:
1. Зашифруйте свой файл локально на вашем компьютере:
   ```bash
   python scripts/prepare_remote_upload.py --input result.json --output result.enc
   ```
2. Скрипт сгенерирует безопасный AES-256 ключ и выведет его в консоль. **Скопируйте этот ключ.**
3. На удалённом сервере запустите backend, предварительно передав ключ через переменную окружения:
   ```bash
   export CHATSTAT_DECRYPT_KEY="<ваш_скопированный_ключ>"
   uvicorn backend.main:app --reload
   ```
4. Откройте интерфейс на удалённом сервере и загрузите полученный `result.enc`. Файл расшифруется строго в оперативной памяти во время анализа.

> **⚠️ ВНИМАНИЕ: ОГРАНИЧЕНИЯ БЕЗОПАСНОСТИ**
> Данная схема защищает файл от перехвата в сети и от утечек на диске. Однако, это **не защищает** от оператора арендованного сервера, который имеет root-доступ и может намеренно инспектировать RAM (оперативную память) процесса во время анализа. Используйте только проверенные хосты с высокой репутацией, если конфиденциальность переписки для вас критична.
