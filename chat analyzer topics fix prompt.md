# ТЗ #4: Фикс тематической кластеризации — мусорные сообщения убили темы

## Контекст

После апгрейда на BERTopic (ТЗ #2) результат стал **хуже**, а не лучше: один кластер забрал
72% всего чата (28421 из 39350 сообщений), назван LLM-ем как "Блядишка" с примерами сообщений
"Так стоп", "Ты чат очистил?", "Зачем". Остальные 4 кластера — "Дозволи" (примеры: "Да", "Да",
"Да"), "Бляфорема" (примеры: "Бля", "Бля", "а"), "Сестры" (примеры: "Брат", "Брат", "Брат"),
"Бля" Слова" (примеры: "бля", "бля", "бля").

Это не тематический анализ — это кластеризация шума. LLM (Ollama) сработала корректно и дала
ровно то название, которое можно вывести из входных данных "Бля, Бля, а" — претензий к LLM
нет, проблема в данных, которые до неё дошли.

Также обнаружена утечка мусора в семантические находки: в "Словарь пары" и "Кого обсуждали"
попали `youtube com`, `https youtube`, `com shorts`, `ассалам алейкума` — нетронутые URL и
бытовые фразы поверх результата.

Это итерация поверх существующего кода. Правь точечно.

## Причина проблемы

1. **Короткие/малосодержательные сообщения** ("Да", "Брат", "а", "Бля") составляют большую
   часть чата (ранее на дашборде было видно 19-25% однословных сообщений). У коротких
   реактивных реплик эмбеддинги получаются похожими друг на друга вне зависимости от реального
   смысла — отсюда гигантский слипшийся кластер.
2. **Мат и эмоциональные междометия** ("бля", "нихуя") не входили в стоп-лист для тематики
   (только частицы вроде "да/не/что" были там по итогам прошлого фикса) — они остались в
   текстах, которые видит BERTopic, и доминируют в TF-IDF-тегах кластеров, потому что очень
   частотны.
3. **URL и стандартные фразы не вычищаются** перед тем, как сообщение идёт в "Словарь пары"
   (п.50) и "Кого обсуждали" (п.37) — `youtube.com`, `https://`, исламские приветствия
   ("ассалам алейкум") пролезают как "уникальный жаргон пары", хотя это типичные шаблонные
   фразы, которые есть у миллионов пользователей.

## БАГ #7: Полностью исключить короткие сообщения из тематической кластеризации

### Требование
Перед запуском BERTopic (и вообще перед любым тематическим анализом — п.33, 34, 17) — отфильтровать
сообщения, которые не несут содержательной темы:

1. Сделай функцию `is_topic_worthy(clean_tokens: list[str], raw_text: str) -> bool` в
   `preprocessing.py`, которая возвращает `False` (сообщение **не участвует** в построении тем),
   если **любое** из условий верно:
   - после очистки (`get_clean_tokens`, с уже расширенным стоп-листом — см. БАГ #8 ниже)
     осталось **меньше 3 содержательных токенов**
   - исходное сообщение (до очистки) состоит из **1-2 слов** общим объёмом
   - сообщение целиком совпадает с типичной реактивной репликой: да, нет, ладно, окей, ок,
     неа, ага, угу, бля, нихуя, и т.п. (используй уже существующий стоп-лист + список
     "реактивных реплик" — сделай его отдельной константой `REACTIVE_PHRASES` в `stopwords.py`,
     не смешивай с обычными стоп-словами, т.к. они нужны для разных целей: стоп-слова
     вычищаются из текста, а реактивные фразы **исключают всё сообщение целиком**)
2. Сообщения, не прошедшие `is_topic_worthy`, **полностью исключаются** из:
   - входного списка текстов для `BERTopic.fit_transform()`
   - расчёта "динамики тем" (п.34)
   - "Словарь пары" (п.50), "фирменные фразочки" (п.36), "Кого обсуждали" (п.37)
3. **Эти же сообщения продолжают участвовать** во всей остальной статистике, где это уместно:
   структурные метрики (1-25), счётчик однословных сообщений (11), счётчик слов-паразитов,
   счётчик мата (см. БАГ #8), sentiment-анализ (28) — короткие эмоциональные реплики тоже несут
   тональность ("бля" в негативном контексте — это сигнал для sentiment), их не нужно убирать
   оттуда, только из ТЕМАТИКИ
4. После фильтрации — если отфильтровано больше 60% сообщений, явно показать это в UI рядом с
   разделом "Темы разговоров": "Темы построены на основе N содержательных сообщений из M общих
   (остальные — короткие реакции, не несущие отдельной темы)" — прозрачность важнее, чем
   замалчивание того, что бо́льшая часть чата — короткие реплики

## БАГ #8: Расширить стоп-лист для тематики — мат и эмоциональные выкрики

### Требование
1. В `stopwords.py` добавь **отдельную категорию** `PROFANITY_AND_EXCLAMATIONS` — мат и
   эмоциональные межометия (бля, нихуя, блять, ёбана, охуеть, пиздец, и т.п. — стандартный
   список русского мата и его сокращений/вариаций типа "бл", "нх"). Сделай список достаточно
   полным, чтобы покрыть частые варианты написания (с учётом несклоняемых форм и сокращений).
2. Эта категория исключается **только из текста, который видит тематический модуль**
   (BERTopic, TF-IDF теги кластеров, "Словарь пары", "Кого обсуждали", "фирменные фразочки") —
   **не убирай её** из общего текста, который видит sentiment-анализ или общая статистика слов,
   так как частота мата сама по себе значимая метрика (п.32 из исходного ТЗ).
3. Добавь отдельную, явно показанную метрику **"Частота мата"** — счётчик употреблений из
   `PROFANITY_AND_EXCLAMATIONS` по каждому автору + общий счётчик (это уже частично есть как
   "Слов паразитов: 1218" на дашборде, но сейчас непонятно, входит ли туда мат или это разные
   категории) — **разведи на дашборде явно две раздельные карточки**: "Слов-паразитов
   (да/нет/ну/типа и т.п.): N" и "Матных слов: M", не смешивай их в одну цифру, т.к. это
   семантически разные вещи и пользователю интересно видеть оба отдельно.
4. Убедись, что функция `get_clean_tokens` (единая точка фильтрации из БАГа #1) принимает
   параметр, какие категории стоп-слов применять — например:
   ```python
   def get_clean_tokens(text: str, exclude_categories: list[str] = None) -> list[str]:
       # exclude_categories может быть ["stopwords", "profanity"] для тематики
       # или просто ["stopwords"] для обычной статистики слов
   ```
   Так у тебя останется одна функция (не плодим дублирующиеся токенизаторы — это и было
   причиной БАГа #1 в прошлый раз), но с гибким поведением под разные нужды.

## БАГ #9: Очистка URL и шаблонных фраз перед "Словарь пары" / "Кого обсуждали"

### Требование
1. Перед любым анализом, который ищет "уникальные"/"специфичные для пары" слова и фразы
   (п.50 "Словарь пары", п.37 "Кого обсуждали", п.36 "фирменные фразочки") — обязательно
   прогонять текст через очистку URL: регулярка для `https?://\S+`, `www\.\S+`, а также
   отдельно удалять остатки доменных слов, которые могут разбиться на токены при токенизации
   (`com`, `youtube`, `shorts`, `http`, `https`, `www`) — добавь их в стоп-лист **именно для
   категории "технический мусор"** (отдельная категория в `stopwords.py`, аналогично БАГу #8)
2. Добавь в список общеязыковых шаблонных фраз (которые не должны считаться "уникальным
   жаргоном пары", даже если часто встречаются) стандартные приветствия/обращения, не
   специфичные для конкретной пары: "ассалам алейкум" и вариации, "доброе утро", "спокойной
   ночи", и подобное — если эти фразы оказываются в топе п.50, это сигнал, что фильтр неполный,
   а не что это действительно уникальный жаргон
3. **Принцип для п.50 "Словарь пары"**: финальный список перед отображением в UI должен
   проходить через фильтр здравого смысла — исключать любые n-граммы, содержащие токены из
   категорий `stopwords`, `profanity` (если это разрешено отдельно от темы — см. ниже),
   `technical_garbage`. Мат **можно** оставлять в "Словаре пары" если он действительно
   специфичен для этой переписки (это релевантно для "забавных находок", не для серьёзной
   тематики) — то есть фильтрация мата в БАГе #8 касается именно темообразующего модуля
   (BERTopic), а не раздела "Забавные находки", где мат к месту и ожидаем пользователем

## БАГ #10: "Кого обсуждали" путает реальные имена с обращениями

### Симптом
В "Кого обсуждали" попало: "Роналду, В, Реала, Я, Реал" — "В", "Я" это явно не имена людей
(предлог и местоимение, захваченные NER-эвристикой по капитализации, видимо, в начале
предложения после точки).

### Требование
1. В NER-эвристике (п.37, "капитализированные слова не в начале предложения") добавь минимальную
   длину токена — **исключай слова короче 3 символов** из списка кандидатов в имена (это
   убирает "В", "Я", "Он" и подобные)
2. Добавь чёрный список однобуквенных/служебных слов, которые могут писаться с большой буквы
   по разным причинам (начало реплики, обращение): "Я", "Ты", "Он", "Мы", "Вы", "А", "И", "В",
   "С", "У", "О" — исключай их явно независимо от длины-фильтра выше, для надёжности
3. После фильтрации — учти падежи: "Реала" и "Реал" — это одно и то же имя в разных падежах,
   но NER-эвристика без лемматизации видит их как два разных слова. Если используешь только
   эвристику по капитализации (без полноценной NER-модели) — добавь лемматизацию и к этому
   списку, чтобы "Реала"/"Реалу"/"Реал" сворачивались в одну именную форму перед подсчётом
   частоты и попаданием в топ

## Проверка перед сдачей

1. Пересчитай тот же тестовый чат (~39 350 сообщений). Убедись, что:
   - Ни один кластер темы не превышает разумную долю чата (если один кластер всё равно
     получается >40% — это сигнал, что фильтрация неполная, нужно перепроверить пороги
     в `is_topic_worthy`)
   - Названия кластеров (через LLM или топ-слова) представляют реальные смысловые темы, а не
     "Да", "Бля", "Брат" в каком-либо виде
   - Примеры сообщений внутри каждого кластера — содержательные предложения, не однословные
     реакции
2. "Словарь пары" и "Кого обсуждали" — проверь глазами, что там нет URL-фрагментов
   (`youtube`, `com`, `https`), нет однобуквенных слов, нет дублей одного имени в разных падежах
3. На дашборде должны появиться раздельно: "Слов-паразитов: N" и "Матных слов: M" (две разные
   карточки, не одна смешанная цифра)
4. Если после фильтрации тематический анализ всё равно даёт малоинформативные кластеры (это
   возможно, если содержательных сообщений в чате реально мало по сравнению с эмоциональными
   репликами) — это честный результат, не нужно искусственно "натягивать" темы там, где их
   мало; в этом случае просто корректно показать предупреждение из БАГа #7 о доле
   отфильтрованных сообщений, чтобы пользователь понимал природу результата
